# Working rules

## Do not code without an explicit go-ahead

**Never write, edit, or commit code until the user explicitly says to start**
("start coding", "start the project", "go ahead and build it", or equivalent).

- A feature request, an idea, or a description of what they want is NOT a
  go-ahead — it is the start of a conversation. Discuss first: restate the
  plan, ask the load-bearing questions, and wait.
- While discussing, reading files is fine. Writing files is not.
- When the go-ahead comes, say what you are about to build before building it.
- This repo is a movie-generation platform and the projects in it are
  creative work: design decisions (what things look like, what exists, how
  the story works) belong to the user, not to whoever writes code fastest.
